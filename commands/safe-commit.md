# /safe-commit

커밋을 안전하게 수행한다. 변경된 테스트 파일이 있으면 test-reviewer로 감사 후 커밋한다.

## 워크플로우

1. `git diff --cached --name-only`로 staged 변경 파일을 확인한다.
2. staged 파일 중 테스트 파일(`*.spec.ts`, `*.test.ts`)이 있는지 확인한다.
3. **테스트 파일이 있으면**:
   - 변경된 각 테스트 파일에 대해 `test-reviewer` 에이전트를 호출한다.
   - 리포트를 출력한다.
   - CRITICAL 항목이 있으면: "CRITICAL 항목이 N개 있습니다. 그래도 커밋할까요?" 라고 사용자에게 확인한다. 사용자가 거부하면 커밋하지 않는다.
   - CRITICAL 없으면: 커밋을 진행한다.
4. **테스트 파일이 없으면**: reviewer 없이 바로 커밋을 진행한다.
5. 커밋 메시지는 staged 변경 내용을 분석해 작성한다 (기존 커밋 스타일 따름).
